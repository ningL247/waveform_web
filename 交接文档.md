# 波形上位机 —— 项目交接文档

> 网页版上位机:把装置经蓝牙发来的微弱生理脉动(足背动脉脉搏)实时显示、滤波、提取、录制。
> 最后更新:2026-06-03。配套文档:[`梳状提取-改进方向.md`](梳状提取-改进方向.md)。

---

## 1. 系统架构

```
装置(STM32F103 + AD7606)
   │  JDY-31 蓝牙(经典SPP)发 FireWater 文本
   ▼
电脑配对出的 COM 口
   │  server.py 读串口 → 解析 → 带通滤波 + 谐波梳状提取
   ▼  WebSocket(同一WiFi)
手机/电脑浏览器(static/index.html):波形 + 频谱 + 提取 + 录制
```

> 为什么要"桥接端":浏览器 Web Bluetooth 连不了 JDY-31 这类经典蓝牙 SPP(iOS 直接锁死),所以由电脑读串口再用 WebSocket 转发,手机/电脑同 WiFi 打开网址即可,无需 App。

---

## 2. 硬件 / MCU 固件状态(重要)

- **板子**:STM32F103 + AD7606(8 通道 16bit ADC,SPI),只用**通道 0**;蓝牙 JDY-31。
- **Keil 工程**:`...\MCU集成、蓝牙、滤波代码\AD7606资料-V0.9-2.0 - 5HZ滤波\AD7606资料-V0.9\驱动代码\数采板-AD7606(KW)-V0.9-SPI-5V\...\USER\TIMER.uvprojx`
- **当前固件**:**均匀 250Hz**(实测 248)、**纯原始单通道**(ch0,单位 mV)、**零片上滤波**(滤波全交给上位机,科研保真)。
- **数据通路**(别乱动):`TIM4` 中断 `ad7606_IRQSrc` 以 250Hz 填 `g_tAD.usBuf`;主循环 `if (g_tAD.usWrite >= CH_NUM)` 每拍发一次。
  - ⚠️ **不要重写这个中断/FIFO**:之前改成 FIFO 架构会让中断不触发、没数据。原始备份在 `USER/*.bak`、`HARDWARE/AD7606/*.bak`。
- **输出格式**:FireWater 文本,`printf("%.2f\n", ch0_mV)`,~250Hz;波特率 **115200**。
- 开机会打印 `Range=0,OS=0` + `1`(一次性横幅,无害)。
- **历史坑**:原固件名义 1000Hz 但靠 `delay_ms(1)`+阻塞 printf 自由跑,实际 ~540Hz 且非均匀,片上"5Hz"低通因 `SAMPLE_FREQ` 写错实际截止 ~2.7Hz。已全部修正为定时器严格 250Hz + 去片上滤波。

## 3. 上位机 server.py

| 模块 | 作用 |
|------|------|
| `Preprocessor` | 因果 Butterworth **带通**(默认 0.5–8Hz,4 阶),逐样本 `sosfilt`;`fs` 自动测量或 `--fs` 钉死 |
| `CombExtractor` | **谐波梳状提取**(弱周期信号):去基线→Welch 估心率→k·f0 窄带之和。详见梳状文档 |
| `SyncAverager` | **同步平均**(方向1,弱周期降噪):在提取输出上检测心拍→相位域 EWMA 平均模板→相位锁定(PLL)重建一条干净「平均脉搏波」。噪声按 ~√N 衰减。详见梳状文档 §3 方向1 |
| `Hub` | 共享状态、待推帧、录制 |
| `FireWaterParser` | 字节流按 `\n` 分帧、`,` 拆多通道浮点 |
| `serial_reader` / `csv_replay` | 真实串口 / 演示回放两种数据源 |
| `ws_handler` / `flusher` | WebSocket 收命令 + ~30fps 批量推送 |

- **每帧** = 原始 + 滤波后 + 提取 + 平均 四组;通道名 = `chX` / `chX(滤波)` / `chX(提取)` / `chX(平均)`。
- **WebSocket 消息**:`meta`(通道名)、`data`(帧)、`rec`(录制状态)、`filter`(滤波设置)、`extract`(提取设置+心率)、`average`(同步平均设置+已平均拍数)。**命令**:`record_start/stop`、`filter`、`extract`、`average`。
- **命令行参数**:`--serial COM --baud --demo CSV --rate --loop --names --fs --low --high --order --no-filter --no-extract --harm --bw --no-avg --avg-alpha --host --port`。

## 4. 网页 static/index.html

- **显示**:上=实时波形(Canvas),下=频谱面板(按 `📊 频谱` 开关,**0–15Hz**,自动标心率主峰 bpm)。频谱 **点数(NFFT)可调** 512–8192,旁边实时显示频率分辨率 Δf=fs/NFFT(点数越大越细但需更多缓冲样本)。
- **控件**:暂停 / 自动量程 / 频谱 / 录制 / 窗口点数 / 采样率显示 / 带通滤波(低·高·阶) / **提取(+心率显示)** / **同步平均(+已平均拍数显示)**。
- **默认**:隐藏"原始",显示**滤波(橙)+ 提取(绿)+ 平均(粉)**三条;点图例可切换任意通道。
- WebSocket 自动重连;HTML 每次请求实时读盘 → **改网页只需刷新,不用重启 server**。

## 5. 数据格式

- **串口(FireWater)**:每行一帧,多通道逗号分隔,`\n` 结尾。如 `955.21\n`。
- **录制时间序列** `recordings/rec_<时间>.csv`:表头 `ch0,ch0(滤波),ch0(提取),ch0(平均)`,逐行四列。
- **录制频谱** `recordings/rec_<时间>_spectrum.csv`:`freq_Hz, ch0_psd, ch0(滤波)_psd, ch0(提取)_psd, ch0(平均)_psd`(各列 Welch 功率谱)。录制存**原始**(滤波/提取/平均为附带),科研保真。

## 6. 怎么运行

1. **真实硬件**:双击 `启动-真实硬件.bat`(已设 `COM=COM8`、`BAUD=115200`、`--fs 250`、自动清理占用端口、等服务起来再开浏览器)。
2. **演示(不接硬件)**:双击 `启动-演示模式.bat`(回放 `data/` 里的 CSV)。
3. 网页:本机 `http://localhost:8090`;手机 `http://<电脑局域网IP>:8090`(同 WiFi,防火墙放行 8090)。
4. **录制**:网页点 `● 录制` → `■ 停止`,在 `recordings/` 生成时间序列 + 频谱两个 CSV。

> 改了 server.py 要**重启 bat 生效**;改了 index.html **刷新网页**即可。

## 7. 关键经验 / 踩过的坑(务必看)

1. **.bat / .c 必须 GBK 编码**:中文系统 cmd 按 GBK 读 .bat,存成 UTF-8 会乱码、解析错乱、server 起不来。`index.html`/`.md` 是 UTF-8。改这些文件要保持 GBK(用脚本读 GBK→替换→写 GBK)。
2. **端口 8090 被旧进程占用** → 闪退;bat 启动前已用 PowerShell 自动结束占用进程。
3. **浏览器在数据到达前连接** → 只拿到兜底单通道 `ch0`、画的是原始列(看着"没滤波/很毛")。已修:`flusher` 在通道名确定后**主动广播 meta**。症状复现时:刷新网页,确认图例有 3 条。
4. **fs 要钉死 `--fs 250`**:自动测 fs 在蓝牙突发时可能偏,导致滤波通带漂移。固件既然是精确 250,就钉死。
5. **JDY-31 需复位 MCU 才开始发数据**;灯**常亮=已连**、闪烁=未连;`VOFA+` 与 server **不能同时占同一 COM 口**。
6. **COM 口**:`启动-真实硬件.bat` 用 **COM8**;反复配对会让蓝牙 SPP 口号变,没数据时去「设置→蓝牙→更多蓝牙设置→COM 端口」查"传出"口,改 bat 里 `set COM=`。
7. **弱信号是应用常态**:足背脉动本就微弱、埋在噪声里,目标是**用算法提取**,不要误判成接触故障(虽然接触好能显著提高原始 SNR)。

## 8. 已知问题 / 待办

- **提取启动延迟**:心率锁定需 ~8–16s 缓冲;开头 ~20s 可能短暂锁错(随后每 0.5s 自动纠正)。弱窄带提取固有。
- **同步平均(方向1)已落地**:第 4 路 `chX(平均)`,需 ≥3 拍才出波(约 2–3s),实时 EWMA(α=0.2)≈ 9.5dB 降噪。离线在 `data/ch0.csv` 验证:561 拍、分半相关 r=0.998。下一步可做**方向4 谐波自动取舍**(几乎零成本)或**方向2 更窄齿+跟踪**。
- **提取后仍有残留杂波** → 看「平均」那条更干净;进一步改进路线见 [`梳状提取-改进方向.md`](梳状提取-改进方向.md)。
- 开机横幅 `Range=0,OS=0`/`1` 会给数据流混入个别杂样本(无害,可在 MCU main.c 删那两行 printf)。
- 频谱面板用单次 FFT(显示用),宽窗口时可能有轻微抽样混叠(非数据问题)。
