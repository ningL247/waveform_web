# 波形上位机(网页版)

手机/电脑浏览器打开即可显示波形。装置经 **JDY-31 蓝牙(经典 SPP)** 发数据。

## 为什么需要「桥接端」
浏览器(Web Bluetooth)只能连 BLE,**连不了 JDY-31 这种经典蓝牙 SPP**(iOS 更是直接锁死)。
所以由一台电脑把 JDY-31 配对成串口读数据,再用 WebSocket 转发给网页:

```
装置 ──蓝牙SPP──> 电脑(server.py 读COM口)──WiFi/WebSocket──> 手机&电脑浏览器
```

手机和电脑只要在**同一个 WiFi** 下,打开网址即可,无需任何 App。

## 安装
```
pip install -r requirements.txt
```

## 用法

### 1) 演示模式(先看界面,不接硬件)
```
python server.py --demo "D:\DesktopD\data\data\1zhongjiawen.csv" --rate 500 --loop
```

### 2) 真实模式(接 JDY-31)
先在 Windows「蓝牙和其他设备」里配对 JDY-31,记下分配的**传出 COM 口**(如 COM5),然后:
```
python server.py --serial COM5 --baud 115200
```
> JDY-31 默认波特率常见为 9600,如果没数据就把 `--baud` 改成 9600。

### 打开网页
- 本机:  http://localhost:8080
- 手机:  http://<电脑局域网IP>:8080  (本机 IP 例如 `172.20.11.18`)

> 手机连不上时,在电脑防火墙放行 8080 端口(或临时关闭专用网络防火墙)。

## 参数
| 参数 | 说明 |
|------|------|
| `--serial COM5` | 蓝牙串口/COM 口 |
| `--baud 115200` | 波特率 |
| `--demo 路径.csv` | 回放 CSV(演示) |
| `--rate 500` | 演示回放速率(样本/秒) |
| `--loop` | 演示循环回放 |
| `--port 8080` | 服务端口 |

## 数据格式(VOFA+ FireWater)
串口上每行一帧文本,多通道用逗号分隔,`\n` 结尾:
```
1048.27\n              ← 单通道
1.2,3.4,5.6\n          ← 三通道
```
网页会自动按通道数画多条曲线。

## 网页操作
- **暂停/继续**、**自动量程**(关掉后用固定量程)
- **窗口**:显示最近多少个点
- 实时显示采样率与各通道当前值
