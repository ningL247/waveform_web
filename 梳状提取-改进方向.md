# 谐波梳状提取 —— 交接说明与改进方向

> 目标:从噪声中提取**微弱的准周期生理脉动**(足背动脉脉搏,信号弱是应用本身的特性,不是接触故障)。
> 已知:即使时域看着像噪声,频谱里**心率基频通常可检测**(实测 rec_20260602:1.34Hz/81bpm,基频 SNR≈12)。
> 思路:脉动=离散谐波谱线,噪声=宽带连续谱 → 用**锁定心率的谐波梳状**只留谐波、扔掉谱线之间的宽带噪声(比固定带通强)。

---

## 1. 当前实现(已上线)

代码:`server.py` 里的 `CombExtractor` 类;前端 `static/index.html`「提取」按钮 + 心率显示;输出为第 3 路通道 `chX(提取)`。

**处理管线**(`CombExtractor.process`):
```
原始 → 高通 0.5Hz(去基线/DC) → 谐波梳状(∑ k·f0 窄带带通) → 提取脉动
```

**基频 f0(心率)估计**(`_estimate`,每 ~0.5s 一次):
1. 取最近 **16s** 去基线信号(滚动缓冲)。
2. **Welch 平均谱**(分段 8s 各做 FFT 再平均,压噪声方差,真周期峰才稳)。
3. 在 **[0.9, 2.2Hz](54–132bpm)** 内取最强峰 = f0(下限排除呼吸/基线,上限排除高频伪迹)。
4. f0 变化 >0.03Hz 才重建梳状(避免频繁重建暂态)。

**梳状构造**(`_build_comb`):中心在 k·f0(k=1..`n_harm`)、每齿宽 `bw`(±bw/2)的二阶 Butterworth 带通各一,逐样本因果滤波后求和。

**关键参数(默认值)**:
| 参数 | 默认 | 含义 | CLI |
|------|------|------|-----|
| `n_harm` | 5 | 谐波个数 | `--harm` |
| `bw` | 0.4Hz | 每条梳齿带宽(±0.2Hz) | `--bw` |
| `hp` | 0.5Hz | 去基线高通 | — |
| `hr_lo/hr_hi` | 0.9/2.2Hz | 心率搜索带 | — |
| 缓冲 | 16s | f0 估计窗 | — |
| Welch nperseg | 8s | 分段长 | — |
| enabled | True | 开关 | `--no-extract` |
| `fs` | (跟随 preproc) | 采样率,实机钉死 250 | `--fs` |

**验证方法**:demo 回放真实数据查 HR 是否锁对:
```
python server.py --demo data/foot_rest_20260601_224304.csv --rate 250 --loop --fs 250 --port 80xx
```
预期稳态:foot_rest→~95bpm;rec_20260602(取原始列)→~81bpm。

## 2. 当前局限 / 残留杂波来源

1. **梳齿有宽度(±0.2Hz)**:缝内的噪声会漏过来。齿越窄越干净,但跟不住心率变化。
2. **心率会抖(HRV)**:f0 漂移时能量泄到齿外,所以齿不能太窄 → 必然漏入一些噪声。
3. **正好落在谐波频率上的噪声去不掉**(例:rec 数据里 ~5.37Hz 的强伪迹)。
4. **弱高次谐波拖后腿**:第 5 次谐波 SNR 只有 ~0.8,纳入它反而把噪声带进来。
5. **启动延迟 ~8–16s**:f0 估计要 ≥8s 缓冲;且开头传感器/频谱未稳时可能短暂锁错(随后每 0.5s 更新自动纠正)。这是弱窄带提取的固有代价。

## 3. 改进方向(按性价比排序)

### ★ 方向 1:同步平均(coherent / ensemble averaging)—— ✅ 已落地(2026-06-03)
- **思路**:用梳状输出当触发,检测每个心拍 → 对齐 → 对最近 N 拍做滑动/指数加权平均。噪声按 **√N 衰减**(离线那条超干净单拍就是这么来的)。
- **已实现**:`server.py` 的 `SyncAverager` 类;输出第 4 路 `chX(平均)`;前端「同步平均」按钮 + 已平均拍数显示;CLI `--no-avg --avg-alpha`。
  - **管线**:在提取输出上 `find_peaks` 定位收缩峰(距 0.6P、prominence 0.3σ)→ 截取心拍窗 `[峰-0.3P, 峰+0.7P]` → 重采样到 256 点的**相位域**(故对 HRV/心率漂移天然鲁棒)→ **去线性趋势使首尾相等**(相位环 1→0 连续,消除每拍跳变)→ EWMA 进模板(α 默认 0.2,等效平均 ~9 拍)。
  - **实时重建**:按心率相位锁定播放模板,每 0.3s 用最近真实峰**渐进对齐相位(PLL,误差摊到区间内)**而非硬跳,避免波形跳变;模板线性插值。
  - **就绪条件**:≥3 拍(约 2–3s)才出波,之前输出 0。
- **离线验证**(`offline_validate_avg.py`,`data/ch0.csv`,5.9min):HR 90bpm、561 拍、**分半相关 r=0.998**、粗糙度↓63%、相干能量占比 51%;集成理论增益 27.5dB,实时 EWMA(α=0.2)≈ 9.5dB。
- **收益**:对弱周期信号收益最大,波形最干净。**代价/局限**:平掉逐拍差异(只要干净代表波形则无妨);依赖心拍检测稳健性;启动需几拍。
- **可调**:`--avg-alpha` 越小越平滑/收敛越慢(N_eff≈(2-α)/α);齿/峰阈值在 `SyncAverager` 内。

### 方向 2:更细心率跟踪 + 更窄梳齿
- **思路**:用锁相环(PLL)/ Kalman 持续精确跟踪 f0,使梳齿可以**更窄**(漏入噪声更少)同时仍跟得住 HRV。
- **实现**:把"每 0.5s 重估 + 0.03Hz 阈值"升级为连续相位跟踪;`bw` 可降到 ~0.2Hz。
- **收益**:更干净、延迟低。**代价**:跟踪环不稳时会抖;实现略复杂。**工作量**:中。

### 方向 3:维纳加权梳状(SNR 加权)
- **思路**:梳齿内不再"矩形全通",而是按各频点估计的信噪比加权(维纳增益 = S/(S+N)),强谐波保留、缝内噪点压低。MSE 最优。
- **实现**:估计每齿内的信号/噪声功率(噪声可用齿外邻域插值),对 FFT 系数加权后重构;或滑窗 FFT 实现。
- **收益**:在不缩窄齿宽的前提下进一步降噪。**代价**:需可靠的噪声谱估计。**工作量**:中。

### 方向 4:自动取舍 / 加权谐波
- **思路**:只保留 SNR 超阈值的谐波,按 SNR 加权;丢掉像第 5 次(SNR<1)这种只带噪声的谐波。
- **实现**:`_build_comb` 时对每个 k·f0 估计谐波 SNR,低于阈值不建该齿或降权。
- **收益**:小而稳的提升,易做。**代价**:几乎无。**工作量**:小(最快见效)。

### 方向 5:谐波振荡器模型(Kalman)
- **思路**:把脉动建模为"基频+谐波、幅度/相位随时间慢变"的状态空间,逐样本 Kalman 跟踪 → 相当于一个**自适应极窄梳状**。
- **收益**:延迟低、抑噪强、对 HRV 自适应。**代价**:实现最复杂、需调协方差。**工作量**:大。

### 方向 6:根上的办法(最治本)
- 弱信号 SNR 随观测时间 **√T** 提升;而**改善传感器接触/光路/模拟前端**提升原始 SNR,比任何后端算法都管用。
- 也可在 MCU/采集端提高有效位数、加硬件低噪声前置。

### 方向 7:SSA(奇异谱分析,子空间法)
- **思路**:对滑窗做 SSA 分解,保留"心跳子空间"的成分。数据驱动,不强依赖严格谐波结构(对波形畸变/HRV 更宽容)。
- **代价**:计算量比梳状大;需自动分组成分。**工作量**:中。

## 4. 建议路线

1. ~~先做方向 1(同步平均)~~ **✅ 已完成**:网页第 4 路「平均脉搏波」,见上文 §3 方向1。
